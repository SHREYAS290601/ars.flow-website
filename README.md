# ARS.FLOW Website

`website/` is the public-facing product for discovering and evaluating verification-first AI skills.

It intentionally stays separate from `skills-catalog/`, which remains the source-of-truth for skill authoring and registry generation.

## Tech stack

- Next.js 15 (App Router) + TypeScript
- Tailwind CSS
- Zod runtime validation for registry data
- `next-themes` for light/dark persistence
- Vitest (unit tests)
- Playwright (core browse e2e flow)

## Quick start

```bash
pnpm install
pnpm dev
```

Open `http://localhost:3000`.

## Registry source configuration

The app can load registry data from:

1. Local mock: `public/registry/skills.json` (default)
2. Remote endpoint: set `SKILLS_REGISTRY_URL`

Example:

```bash
SKILLS_REGISTRY_URL=https://raw.githubusercontent.com/SHREYAS290601/ars.flow-skills/main/registry/skills.json pnpm dev
```

Data is validated with Zod before rendering. Validation or fetch errors render a clear fallback UI instead of crashing routes silently.

## Scripts

- `pnpm dev` - start local dev server
- `pnpm dev:reset` - clear `.next` cache and start dev server
- `pnpm build` - production build
- `pnpm start` - run production server
- `pnpm lint` - Next.js linting
- `pnpm typecheck` - TypeScript checks
- `pnpm test:unit` - Vitest unit tests
- `pnpm test:e2e` - Playwright smoke/e2e tests
- `pnpm test:all` - typecheck + unit + e2e
- `pnpm reinstall:pnpm` - clean reinstall for package-manager consistency

## Architecture notes

- The website never imports raw skill source files.
- Skill cards, filters, and detail pages render from the validated registry model.
- Category counts are derived from the loaded registry at runtime.
- v1 preview is static and explicitly labeled as validated sample output (no arbitrary code execution).
- Skill docs render in-app via `/skills/[slug]/docs` for reliable access during early catalog bootstrapping.
- Skill downloads are generated by `/api/download/[slug]` as attachment bundles.

## Troubleshooting

If navigation shows a runtime error like `TypeError: __webpack_modules__[moduleId] is not a function`, clear cache and reinstall with a single package manager:

```bash
pnpm run reinstall:pnpm
pnpm dev
```

If you see `Cannot find module './289.js'` from `.next/server/webpack-runtime.js`, run a hard reset:

```bash
pkill -f "next dev" || true
pnpm run reset:hard
pnpm dev
```
